Dasar Web & Pengantar

Jelajahi bagaimana website bekerja, protokol dasarnya, dan konsep inti di balik basis data yang menggerakkan aplikasi modern.

Cara Kerja Website

Website adalah kumpulan halaman web yang diakses daring. Setiap website berada pada server hosting. Peramban mengirim permintaan ke server DNS, yang mengubah URL menjadi alamat IP. Server merespons dengan data, memungkinkan pengguna melihat website.

Proses ini melibatkan *web crawlers* untuk *crawling*, *indexing*, dan *serving search results*.

Protokol HTTP & HTTPS

HTTP adalah protokol dasar untuk transmisi data di internet. Klien mengirim permintaan, server merespons. HTTPS menambahkan lapisan keamanan dengan enkripsi.

  • **GET**: Mengambil data.
  • **POST**: Mengirim data untuk membuat sumber daya.
  • **PUT**: Memperbarui sumber daya.
  • **DELETE**: Menghapus sumber daya.
  • **Caching**: Menyimpan salinan lokal untuk mempercepat pemuatan.

Konsep Dasar Basis Data

Basis data adalah koleksi data terorganisir untuk penyimpanan dan akses efisien. Sistem Manajemen Basis Data (DBMS) adalah perangkat lunak yang berinteraksi dengan basis data.

**Fungsi Utama DBMS:** Definisi Data (DDL), Manipulasi Data (DML), Kontrol Data (DCL), Manajemen Transaksi, Keamanan.

Backend & Basis Data

Perbandingan Bahasa Backend

Setiap bahasa memiliki kekuatan dan kasus penggunaan yang ideal. Pilihan yang tepat bergantung pada kebutuhan proyek Anda.

SQL vs. NoSQL

Database adalah jantung dari aplikasi dinamis. Pahami dua paradigma utama untuk penyimpanan data.

SQL (Relasional)

  • Struktur tabel yang ketat
  • Ideal untuk data terstruktur
  • Skalabilitas vertikal
  • Properti ACID
  • Contoh: MySQL, PostgreSQL

NoSQL (Non-Relasional)

  • Skema yang fleksibel (JSON, dll.)
  • Bagus untuk data tidak terstruktur
  • Skalabilitas horizontal
  • Model Konsistensi BASE
  • Contoh: MongoDB, Redis

Pemodelan Data & Normalisasi (SQL)

Proses membuat representasi data dan mengorganisir tabel untuk minimalkan redundansi dan tingkatkan integritas.

API: Jembatan Antar Aplikasi

API (Antarmuka Pemrograman Aplikasi) memungkinkan komponen perangkat lunak berkomunikasi. RESTful dan SOAP adalah dua gaya arsitektur utama.

RESTful API

  • Gaya arsitektur, berbasis sumber daya.
  • Menggunakan metode HTTP standar (GET, POST, PUT, DELETE).
  • Fleksibel, mendukung JSON (umum).
  • Stateless, Cacheable.
  • Ideal untuk aplikasi publik, mobile, microservices.

SOAP API

  • Protokol ketat, berbasis fungsi.
  • Menggunakan XML.
  • Kaku, server harus mempertahankan state klien.
  • Membutuhkan WS-Security untuk keamanan.
  • Ideal untuk sistem enterprise kompleks (perbankan).

Proses & Praktik

Dari kode ke pengguna. Pelajari tentang alur kerja deployment, kontrol versi dengan Git, dan praktik terbaik untuk keamanan dan kinerja.

Sistem Kontrol Versi: Git & GitHub

Git adalah sistem kontrol versi terdistribusi yang esensial untuk melacak perubahan kode dan kolaborasi tim. GitHub adalah platform hosting Git untuk kolaborasi yang lebih baik.

  • **Manfaat Git:** Pelacakan perubahan, kerja tim efisien (branching), pemulihan mudah, riwayat lengkap.
  • **Perintah Esensial:** `git init`, `clone`, `add`, `commit`, `push`, `pull`, `branch`, `merge`.
  • **GitHub Flow:** Branching, commit, pull request, merge.

Jenis-jenis Hosting Web

Pilihan hosting yang tepat krusial untuk skala, kinerja, keamanan, dan anggaran aplikasi.

Shared Hosting

Banyak situs di satu server. Murah, mudah. Performa terpengaruh situs lain.

VPS Hosting

Server fisik dibagi virtual. Sumber daya dedicated, kontrol lebih. Lebih mahal dari shared.

Dedicated Hosting

Seluruh server untuk satu situs. Performa, kontrol, keamanan tertinggi. Paling mahal.

Cloud Hosting

Jaringan server virtual. Skalabilitas & keandalan tinggi. Fleksibel, aman.

Desain & Arsitektur

Fokus pada pengalaman pengguna (UX) dan struktur tingkat tinggi aplikasi Anda. Keputusan ini memengaruhi bagaimana pengguna berinteraksi dengan produk Anda dan bagaimana produk itu tumbuh.

Prinsip Desain UI/UX & Aksesibilitas

Desain yang baik bukan hanya tentang estetika, tetapi tentang menciptakan pengalaman yang intuitif dan efektif bagi pengguna, serta dapat diakses oleh semua orang.

  • ✓
    Kesederhanaan: Hindari kerumitan, fokus pada esensi.
  • ✓
    Konsistensi: Gunakan elemen dan pola yang sama.
  • ✓
    Hierarki Visual: Pandu mata pengguna ke elemen penting.
  • ✓
    Aksesibilitas (WCAG): Konten dapat diakses semua orang, termasuk penyandang disabilitas (Perceivable, Operable, Understandable, Robust).

Pola Arsitektur Aplikasi

Bagaimana Anda menyusun kode Anda memiliki implikasi besar pada skalabilitas, pemeliharaan, dan kecepatan pengembangan.

Monolitik

Aplikasi tunggal, digabungkan erat. Cepat untuk memulai, sulit untuk diskalakan.

Microservices

Kumpulan layanan kecil dan independen. Fleksibel dan dapat diskalakan, tetapi lebih kompleks.

Serverless

Tanpa manajemen server. Jalankan kode sebagai respons terhadap peristiwa. Hemat biaya, tetapi bisa memiliki *cold start*.

Website Statis vs. Dinamis & SPA vs. PWA

Pahami perbedaan fundamental dalam cara konten disajikan dan interaksi pengguna ditangani.

Website Statis

  • Konten tetap, berubah manual.
  • Interaktivitas terbatas.
  • HTML, CSS, JS sederhana.
  • Tidak perlu basis data.

Website Dinamis

  • Konten berubah otomatis (input pengguna/basis data).
  • Fitur interaktif (login, pencarian).
  • Bahasa sisi server (PHP, Python, Node.js).
  • Terhubung dengan basis data.

Single Page Application (SPA)

  • Muat semua kode awal, update dinamis.
  • Transisi cepat, mirip aplikasi native.
  • Client-side rendering.
  • Sulit untuk SEO.

Progressive Web App (PWA)

  • Teknologi web, pengalaman mirip native.
  • Dapat diinstal, offline, notifikasi push.
  • Baik untuk SEO.
  • Biaya pengembangan lebih rendah.
